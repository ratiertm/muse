# 기능 리서치: AI 캐릭터 챗 앱 핵심 기능

## 1. 주요 경쟁사 분석

### 제타 (Zeta) - 스캐터랩
- 60만+ 캐릭터, 하루 평균 사용시간 2시간 14분
- 국내 1위 AI 캐릭터 챗 앱
- **핵심 기능**: 캐릭터 생성/공유, 커뮤니티, 캐릭터 간 대화 기록 열람, 라이브 스토리

### 크랙 (Crack) - 뤼튼
- 뤼튼의 '캐릭터 챗' 기능 분리 독립 (2025.04)
- Claude Sonnet 3.7 사용 (슈퍼챗)
- **핵심 기능**: 즉흥 서사 플레이, "내 말 한마디에 세계가 갈라진다" 컨셉

### Character.AI
- 글로벌 최대. 수백만 캐릭터
- 자체 LLM 사용
- **핵심 기능**: 그룹채팅, 캐릭터 보이스, 메모리 박스

### SillyTavern
- 오픈소스 로컬 UI. 다양한 LLM 백엔드 연결
- **핵심 기능**: 캐릭터 카드(V2 스펙), World Info/Lorebook, 벡터 메모리, 그룹챗

## 3. 채팅 UI 패턴

### 메시지 포맷
- **대사**: 일반 텍스트 또는 "큰따옴표" 안에
- **행동/묘사**: *이탈릭* (별표)으로 감싸기
- **내면 독백**: (괄호) 또는 별도 스타일링
- **마크다운 지원**: 볼드, 이탈릭, 코드블록

### AI 캐릭터 챗 특화 UI 요소
1. **타이핑 인디케이터**: "세라피나가 입력 중..." + 점 애니메이션
2. **Swipe / 재생성**: 좌우 스와이프로 다른 응답 탐색 (Character.AI 시그니처)
3. **캐릭터 상태 표시**: 온라인/오프라인 (유사 메신저 UX)
4. **반응 이모지**: 메시지에 하트/좋아요 반응
5. **긴 응답 접기**: 매우 긴 응답은 "더보기"로 축소

### 레이아웃
```
┌──────────────────────────┐
│  ← 뒤로  [캐릭터명] ⚙️   │  헤더: 아바타 + 이름 + 설정
├──────────────────────────┤
│                          │
│  [아바타] 캐릭터 메시지    │  메시지 영역
│         *행동 묘사*       │  - 왼쪽: 캐릭터
│         "대사"            │  - 오른쪽: 유저
│                          │
│           유저 메시지 [나] │
│                          │

│         ███████▋         │

```

## 4. 그룹 채팅 구현 (SillyTavern 방식 분석)

### SillyTavern 그룹채팅 아키텍처
1. **응답 순서**: 라운드로빈, 랜덤, 또는 "자연스러운" 순서 선택
2. **캐릭터 카드 합성**: 
   - APPEND 모드: 모든 캐릭터 카드를 하나로 합쳐서 프롬프트에 포함
   - 개별 모드: 응답할 캐릭터의 카드만 포함
3. **자동 모드**: 5초 딜레이 후 다음 캐릭터가 자동 응답
4. **개별 메모리**: SillyTavern-Presence 플러그인으로 각 캐릭터 별도 메모리

### 구현 시 고려사항
- 각 캐릭터의 시스템 프롬프트를 개별 관리
- "누가 다음에 말할지" 결정 로직 (LLM에게 판단 위임 가능)
- 그룹채팅은 토큰 소비가 1:1 대비 2-5배 → 비용 관리 중요
- 캐릭터 간 대화에서 일관성 유지가 핵심 난제
