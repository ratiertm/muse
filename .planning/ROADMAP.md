# ROADMAP.md — Muse

## Phase 1: 백엔드 코어
**Goal: API 서버가 캐릭터 생성하고 채팅 응답을 스트리밍할 수 있다**
- FastAPI 프로젝트 셋업
- DB 스키마 (유저, 캐릭터, 시나리오, 대화)
- 캐릭터 CRUD API
- LLM 연동 (litellm — GPT-4o-mini + Claude Sonnet)
- 기본 채팅 API (1:1, SSE 스트리밍)
- 인증 (심플 PIN/프로필)

**Success: curl로 캐릭터 만들고 채팅 응답이 스트리밍으로 온다**

## Phase 2: God Agent
**Goal: 전지적 오케스트레이터가 세계 상태를 관리하고 캐릭터에게 브리핑한다**
- God Agent 프롬프트 설계
- World State / Knowledge Graph / Private State 모델
- 대화 전 브리핑 파이프라인
- 대화 후 이벤트 추출 + 상태 업데이트
- 시나리오 CRUD API

**Success: 같은 시나리오에서 캐릭터 A와 대화한 내용이 캐릭터 B 브리핑에 반영된다**

## Phase 3: 그룹 채팅
**Goal: 여러 캐릭터가 동시에 대화하며 각자 다른 지식 상태를 유지한다**
- 그룹챗 API (턴 오케스트레이션)
- God Agent 그룹 모드 (턴별 브리핑)
- 캐릭터 간 지식 차이 반영

**Success: 3명 캐릭터 그룹챗에서 비밀을 아는 캐릭터만 반응한다**

## Phase 4: Flutter 앱 — 기본 UI
**Goal: 안드로이드 앱에서 캐릭터 생성하고 채팅할 수 있다**
- Flutter 프로젝트 셋업
- 프로필 선택 화면
- 캐릭터 목록 화면
- 캐릭터 생성 폼
- 1:1 채팅 UI (스트리밍, swipe 재생성)
- 깔끔한 미니멀 디자인 (다크/라이트)

**Success: APK 설치 후 캐릭터 만들고 대화할 수 있다**

## Phase 5: 고급 기능
**Goal: 원작 자동 생성, 아바타, 그룹챗 UI 완성**
- 원작 기반 캐릭터 자동 생성 (작품명+캐릭터명 → AI)
- 애니풍 아바타 AI 생성
- 그룹 채팅 UI
- 시나리오 관리 UI
- 유저 페르소나 설정 UI

**Success: "죠죠 디오" 입력하면 캐릭터가 자동 생성되고 애니 아바타도 나온다**

## Phase 6: 배포 & 폴리싱
**Goal: 실사용 가능한 완성도**
- 오라클 춘천 서버 배포
- APK 빌드 & 설치
- 에러 처리, 로딩 상태
- 오프라인/네트워크 에러 대응
- 성능 최적화 (응답 속도)
- MB + 딸 테스트 & 피드백 반영

**Success: 딸이 매일 쓴다**
