# Plan: 유저 페르소나 & 시나리오 참여 시스템

## 개요
- **Feature**: user-persona-ui
- **요구사항**: REQ-09 (유저 페르소나) 확장
- **목표**: 유저가 페르소나를 만들고, 그 페르소나로 시나리오에 참여하여 가상 캐릭터들과 대화

## 핵심 유저 플로우

```
페르소나 관리 → 시나리오 선택 → 페르소나 + 캐릭터 선택 → 그룹 채팅 → 대화 저장 → 나중에 이어하기
```

### 상세 플로우
1. **내 페르소나 관리**: 여러 페르소나 생성/수정/삭제
   - 예: "쿠조 죠타로", "김독자의 친구", "학생 A"
2. **시나리오 진입 시 페르소나 선택**: 시나리오에 들어갈 때 "나는 누구로 참여할 것인가" 선택
3. **대화할 캐릭터 선택**: 시나리오 내 AI 캐릭터 중 이번 대화에 참여할 캐릭터들 선택
4. **채팅 저장 & 이어하기**: 대화 기록이 저장되어 나중에 같은 시나리오/페르소나/캐릭터 조합으로 이어서 대화

## 기능 요구사항

### FR-01: 페르소나 관리
- 페르소나 목록 화면 (설정에서 진입)
- 페르소나 생성: 이름, 외모, 성격, 설명
- 페르소나 수정/삭제
- 기본 페르소나 설정 (is_default)

### FR-02: 시나리오 참여 시 페르소나 선택
- 그룹 채팅 만들기 화면에서 "내 페르소나" 선택 드롭다운/라디오 추가
- 선택한 페르소나가 God Agent 프롬프트에 주입됨
- 페르소나 미선택 시 기본 페르소나 또는 "User"로 참여

### FR-03: 시나리오 내 캐릭터 선택
- 기존 그룹 채팅 만들기 화면의 캐릭터 선택 (이미 구현됨)
- 시나리오에 연결된 캐릭터 중 대화에 참여할 캐릭터를 체크박스로 선택

### FR-04: 대화 저장 & 이어하기
- 그룹 채팅 대화가 conversation으로 저장됨 (이미 구현됨)
- 시나리오 화면에서 이전 대화 목록 표시
- 이전 대화 터치 시 이어서 대화 가능
- 대화 목록에 참여 페르소나 + 캐릭터 이름 표시

## 현재 구현 상태

| 기능 | 백엔드 | 프론트엔드 |
|------|:------:|:----------:|

| 시나리오 캐릭터 선택 | ✅ | ✅ |
| 그룹 채팅 생성/저장 | ✅ | ✅ |
| 대화 목록 | ✅ | ✅ (캐릭터별) |

## 필요한 변경

### 백엔드
1. **Conversation 모델에 `persona_id` 추가** — 어떤 페르소나로 대화했는지 기록
2. **그룹 채팅 생성 API에 `persona_id` 파라미터 추가**
3. **채팅 시 지정된 persona_id의 페르소나를 프롬프트에 주입** (현재는 default만)
4. **대화 목록 API에 persona 정보 포함**

### 프론트엔드
1. **Persona 모델 (freezed)** — `persona.dart`
2. **PersonaRepository** — CRUD API 연동
3. **PersonaProvider** — Riverpod 상태 관리
4. **페르소나 목록 화면** — 생성/수정/삭제
5. **페르소나 편집 화면** — 폼
6. **그룹 채팅 만들기 화면 수정** — 페르소나 선택 추가
7. **시나리오 대화 목록** — 이전 대화 이어하기
8. **설정 화면** — "내 페르소나" 메뉴 추가
9. **라우터** — 새 화면 라우트 추가

## 성공 기준
1. 페르소나 3개 만들어서 각각 다른 시나리오에 참여 가능
2. "쿠조 죠타로"로 전독시 시나리오에서 김독자, 유중혁과 대화
3. 대화를 끝내고 다시 앱에 들어와도 이전 대화를 이어갈 수 있음
4. 대화 목록에 "쿠조 죠타로 × 김독자, 유중혁" 형태로 표시
